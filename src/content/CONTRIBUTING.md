# Content Trove Contributing Guide

This guide documents how to author content for King's Road -- a pastoral medieval pilgrimage RPG where the player walks the King's Road seeking the Holy Grail.

## Overview

The **content trove** is a collection of JSON files that define every piece of game content: the road spine (world layout), quests, NPCs, road features, and more. These files are validated against Zod schemas at build time and consumed by the game engine at runtime.

Content can be authored by humans or generated by AI agents. Either way, all content must pass schema validation and referential integrity checks before merging.

## Directory Structure

```
content/
├── world/               Road spine and world configuration
│   └── road-spine.json  The 1D road layout with anchor points
├── main-quest/          Main story quests (one JSON per chapter)
├── side-quests/
│   ├── macro/           Long multi-chapter side quest chains
│   ├── meso/            Medium quests (village-scale problems)
│   └── micro/           Short roadside encounters
├── npcs/                NPC definition files (one per archetype)
├── features/            Road features (bridges, shrines, ruins)
└── CONTRIBUTING.md      This file
```

Each JSON file in a content directory must validate against its corresponding Zod schema. The validation script determines which schema to use based on the file's directory location:

| Directory | Schema | Description |
|-----------|--------|-------------|
| `content/world/*.json` | `RoadSpineSchema` | Road spine and anchor definitions |
| `content/main-quest/*.json` | `QuestDefinitionSchema` | Main story quests |
| `content/side-quests/macro/*.json` | `QuestDefinitionSchema` | Long side quest chains |
| `content/side-quests/meso/*.json` | `QuestDefinitionSchema` | Medium village quests |
| `content/side-quests/micro/*.json` | `QuestDefinitionSchema` | Short roadside encounters |
| `content/npcs/*.json` | `NPCDefinitionSchema` | NPC archetype definitions |
| `content/features/*.json` | `FeatureDefinitionSchema` | Roadside feature definitions |

## Schema Reference

### Road Spine (`content/world/road-spine.json`)

Defines the linear 1D road that forms the game's world axis.

**Schema: `RoadSpineSchema`**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `totalDistance` | integer (positive) | Yes | Total road length in world units |
| `anchors` | AnchorPoint[] (min 2) | Yes | Ordered anchor points along the road. First anchor must have `distanceFromStart: 0` |
| `regions` | Region[] | No | Named biome regions spanning anchor ranges |

**AnchorPoint fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string (min 1) | Yes | Unique identifier (e.g., `"home"`, `"anchor-01"`) |
| `name` | string (2-60 chars) | Yes | Display name (e.g., `"Ashford"`, `"Millbrook"`) |
| `type` | enum | Yes | One of: `VILLAGE_FRIENDLY`, `VILLAGE_HOSTILE`, `DUNGEON`, `WAYPOINT` |
| `distanceFromStart` | integer (>= 0) | Yes | Distance from road start in world units |
| `mainQuestChapter` | string (min 1) | Yes | Associated main quest chapter ID |
| `description` | string (10-500 chars) | Yes | Prose description of the location |
| `features` | string[] (min 1) | Yes | Available features at this anchor (e.g., `"tavern"`, `"blacksmith"`) |
| `sideQuestSlots` | integer (>= 0) | No | Number of side quest slots (default: 0) |

**Example:**

```json
{
  "totalDistance": 30000,
  "anchors": [
    {
      "id": "home",
      "name": "Ashford",
      "type": "VILLAGE_FRIENDLY",
      "distanceFromStart": 0,
      "mainQuestChapter": "chapter-00",
      "description": "Your home town, a quiet farming village at the foot of gentle hills.",
      "features": ["home", "tavern", "blacksmith"]
    },
    {
      "id": "anchor-01",
      "name": "Millbrook",
      "type": "VILLAGE_FRIENDLY",
      "distanceFromStart": 6000,
      "mainQuestChapter": "chapter-01",
      "description": "A market town where the King's Road crosses the River Mill.",
      "features": ["tavern", "market", "chapel"]
    }
  ]
}
```

### Quest Definition (`content/main-quest/`, `content/side-quests/{macro,meso,micro}/`)

One JSON file per quest. The quest tier and branching requirements depend on which directory the file is in.

**Schema: `QuestDefinitionSchema`**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string (min 1) | Yes | Unique quest identifier (e.g., `"micro-lost-merchant"`) |
| `tier` | enum | Yes | One of: `macro`, `meso`, `micro` |
| `title` | string (3-100 chars) | Yes | Human-readable quest title |
| `estimatedMinutes` | number (positive) | Yes | Estimated play duration in minutes |
| `anchorAffinity` | string | Yes | ID of the anchor this quest is associated with |
| `trigger` | QuestTrigger | Yes | How the quest is triggered (see below) |
| `steps` | QuestStep[] | Conditional | Linear quest steps (for micro quests) |
| `branches` | `{ A: QuestBranch, B: QuestBranch }` | Conditional | A/B branching paths (required for meso/macro) |
| `prerequisites` | string[] | No | IDs of quests that must be completed first |
| `reward` | QuestReward | Yes | Reward given on completion |

**Quest must have either `steps` or `branches` (or both). Meso and macro quests must have `branches`.**

**QuestTrigger** (discriminated union on `type`):

- `{ "type": "roadside", "distanceRange": [min, max] }` -- triggered by walking within a distance range
- `{ "type": "anchor", "anchorId": "anchor-01" }` -- triggered at a specific anchor
- `{ "type": "prerequisite", "questId": "some-quest" }` -- triggered after completing a quest

**QuestStep fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string (min 1) | Yes | Unique step ID within the quest |
| `type` | enum | Yes | One of: `dialogue`, `fetch`, `escort`, `investigate`, `encounter`, `travel`, `puzzle` |
| `description` | string (max 500) | No | Step description |
| `npcArchetype` | string | No | NPC archetype for dialogue steps |
| `dialogue` | string | No | Actual dialogue text (must meet word count minimums when present) |
| `dialogueMinWords` | integer (positive) | No | Minimum word count for dialogue (default: 15) |
| `dialogueMaxWords` | integer (positive) | No | Maximum word count for dialogue (default: 80) |
| `destination` | string | No | Destination anchor ID for travel/escort steps |
| `itemId` | string | No | Item ID for fetch steps |
| `encounterId` | string | No | Encounter ID for encounter steps |

**QuestBranch fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `label` | string (3-100 chars) | Yes | Player-facing choice label |
| `steps` | QuestStep[] (min 1) | Yes | Steps in this branch |
| `reward` | QuestReward | No | Branch-specific bonus reward |

**QuestReward fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | enum | Yes | One of: `item`, `modifier`, `unlock`, `currency` |
| `itemId` | string | No | Item ID (when type is `item`) |
| `modifierId` | string | No | Modifier ID (when type is `modifier`) |
| `unlockId` | string | No | Unlock ID (when type is `unlock`) |
| `amount` | number | No | Amount (when type is `currency`) |

**Micro quest example (linear, no branches):**

```json
{
  "id": "micro-lost-merchant",
  "tier": "micro",
  "title": "The Lost Merchant",
  "estimatedMinutes": 8,
  "anchorAffinity": "anchor-01",
  "trigger": { "type": "roadside", "distanceRange": [5000, 7000] },
  "steps": [
    {
      "id": "step-01",
      "type": "dialogue",
      "npcArchetype": "merchant",
      "dialogueMinWords": 15,
      "dialogueMaxWords": 80,
      "dialogue": "Good traveler, I have lost my way on these winding paths. The road forked and I foolishly took the wrong turn. Could you kindly escort me to Millbrook? I have wares to sell at the market and the day grows short."
    },
    {
      "id": "step-02",
      "type": "escort",
      "destination": "anchor-01",
      "description": "Escort the merchant to Millbrook."
    }
  ],
  "reward": { "type": "item", "itemId": "merchant-map" }
}
```

**Meso quest example (with A/B branches):**

```json
{
  "id": "meso-poisoned-well",
  "tier": "meso",
  "title": "The Poisoned Well",
  "estimatedMinutes": 25,
  "anchorAffinity": "anchor-01",
  "trigger": { "type": "anchor", "anchorId": "anchor-01" },
  "branches": {
    "A": {
      "label": "Confront the poisoner",
      "steps": [
        {
          "id": "a-01",
          "type": "investigate",
          "description": "Search the well house for clues to the poisoner's identity."
        },
        {
          "id": "a-02",
          "type": "encounter",
          "encounterId": "poisoner-fight"
        }
      ],
      "reward": { "type": "modifier", "modifierId": "village-hero" }
    },
    "B": {
      "label": "Find the cure",
      "steps": [
        {
          "id": "b-01",
          "type": "fetch",
          "itemId": "moonpetal",
          "description": "Find moonpetal herbs growing wild in the forest."
        },
        {
          "id": "b-02",
          "type": "dialogue",
          "npcArchetype": "healer",
          "dialogueMinWords": 15,
          "dialogueMaxWords": 80,
          "dialogue": "You found the moonpetal! Let me brew the antidote right away. The village folk have been suffering something terrible, and this herb is just what we need to set things right."
        }
      ],
      "reward": { "type": "modifier", "modifierId": "village-healer" }
    }
  },
  "reward": { "type": "item", "itemId": "well-keeper-ring" }
}
```

### NPC Definition (`content/npcs/`)

One JSON file per NPC archetype. NPCs are template-based: each archetype has a pool of names and greetings that the game randomly selects from.

**Schema: `NPCDefinitionSchema`**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string (min 1) | Yes | Unique NPC definition ID |
| `archetype` | enum | Yes | One of: `blacksmith`, `innkeeper`, `merchant`, `wanderer`, `healer`, `knight`, `hermit`, `farmer`, `priest`, `noble`, `bandit`, `scholar` |
| `namePool` | string[] (min 3, each 2-40 chars) | Yes | Pool of names the game can assign |
| `greetingPool` | DialogueLine[] (min 2) | Yes | Random greetings (each 10-300 chars) |
| `questDialogue` | Record<questId, DialogueLine[]> | No | Quest-specific dialogue lines |
| `appearance` | `{ clothColor?, accessory? }` | No | Visual appearance hints |

**DialogueLine:** `{ "text": string (10-300 chars), "condition"?: string }`

**Example:**

```json
{
  "id": "npc-merchant",
  "archetype": "merchant",
  "namePool": ["Edwin", "Godric", "Aldric", "Wulfstan"],
  "greetingPool": [
    { "text": "Welcome, traveler! Have a look at my fine wares from across the kingdom." },
    { "text": "Good day to you! The road has been kind today, and so shall I be with my prices." },
    { "text": "Ah, a pilgrim on the King's Road! Come, rest your feet and browse my goods." }
  ]
}
```

### Feature Definition (`content/features/`)

Road features are points of interest placed along the King's Road by the pacing engine.

**Schema: `FeatureDefinitionSchema`**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string (min 1) | Yes | Unique feature ID |
| `tier` | enum | Yes | One of: `ambient`, `minor`, `major` |
| `name` | string (2-80 chars) | Yes | Display name |
| `description` | string (10-300 chars) | Yes | Prose description |
| `visualType` | string | Yes | Renderer key (e.g., `"stone_bridge"`, `"shrine"`, `"ruin"`) |
| `interactable` | boolean | No | Whether the player can interact (default: false) |
| `dialogueOnInteract` | string (max 300) | No | Text shown on interaction |

**Example:**

```json
{
  "id": "feature-wayside-shrine",
  "tier": "minor",
  "name": "Wayside Shrine",
  "description": "A small stone shrine to Saint Cuthbert, adorned with wildflowers left by passing pilgrims.",
  "visualType": "shrine",
  "interactable": true,
  "dialogueOnInteract": "You kneel before the shrine and offer a quiet prayer. The scent of wildflowers fills the air."
}
```

## Tone Guide

King's Road is set in a romanticized, pastoral vision of medieval England. The mood is warm, hopeful, and grounded.

### Do

- **Warm sunlight**, rolling green hills, village markets, pilgrims on the road
- Natural, period-appropriate dialogue that remains readable
- Quiet beauty in everyday things: a well-tended garden, a stone bridge over a brook
- Gentle humor, kindness between strangers, the fellowship of the road
- Mystery and wonder (ancient ruins, holy relics) without horror
- Rich sensory descriptions (sunlight, birdsong, bread baking, meadow flowers)
- Characters who are weary but kind, or gruff but fair
- References to seasons, harvests, pilgrimages, holy days
- Arthurian resonance (Grail quests, noble virtues, chivalry)
- Regional English vocabulary (dale, shire, moor, heath, downs, brook)
- English/Celtic/Norman names

### Do Not

- Grimdark violence, gore, torture, or cruelty
- Modern slang, internet humor, or anachronistic language
- High fantasy tropes (elves, dragons, magic systems, chosen ones)
- Cynical, nihilistic, or edgy themes
- Sarcasm or irony that breaks the pastoral mood
- Names from other fantasy settings (no Tolkien, no anime, no video game references)

### Name Examples

**People:** Edwin, Godric, Aldric, Wulfstan, Eadgyth, Aelfrith, Oswin, Leofric, Mildred, Hild, Cenric, Dunstan

**Places:** Ashford, Millbrook, Thornfield, Ravensgate, Stonebarrow, Hayward's Crossing, Aldbury, Briardale

## Quest Writing Rules

### Branching Requirements

- **Micro quests** can be linear (no branching required). They are short roadside encounters.
- **Meso quests** MUST have A and B branches. These represent genuinely different approaches to solving a problem, not cosmetic differences.
- **Macro quests** MUST have A and B branches. These are major story arcs with significant consequences.

### Branch Quality

A and B branches should:

1. Represent **genuinely different approaches** (e.g., confront vs. negotiate, fight vs. heal, help vs. ignore)
2. Have **different rewards** reflecting the approach taken
3. Feel **equally valid** -- neither branch should be the "obvious right choice"
4. Lead to **different narrative outcomes** where possible

Bad branching (avoid):
- A: "Go left" / B: "Go right" (cosmetic difference only)
- A: "Be good" / B: "Be evil" (moral binary without nuance)

Good branching (aim for):
- A: "Confront the poisoner directly" / B: "Find a cure in the forest" (different skills/approaches)
- A: "Help the merchant recover his goods" / B: "Escort the merchant to safety" (different priorities)

## Validation

Run the trove validation script to check all content:

```bash
npx tsx scripts/validate-trove.ts
```

Add `--verbose` for detailed output:

```bash
npx tsx scripts/validate-trove.ts --verbose
```

### What the Script Checks

1. **Schema validation**: Every JSON file is validated against its Zod schema based on directory location
2. **Referential integrity**:
   - Quest `prerequisites` reference existing quest IDs
   - Quest `anchorAffinity` references existing anchor IDs
   - NPC archetypes used in quest steps exist in NPC definitions
   - Encounter IDs used in quest steps exist in encounter definitions
3. **Duration estimation**: Compares paper-playtested duration to declared `estimatedMinutes` (warns if >50% deviation)
4. **A/B branch coverage**: Meso and macro quests must have both A and B branches
5. **Substance score**: Measures dialogue word density per step and unique NPC count per quest (warns if below threshold)

### Output

The script prints a formatted report to stdout and writes a JSON report to `content/validation-report.json`. Exit codes: `0` for success (may have warnings), `1` if any file fails schema validation.

## Paper Playtesting Format

The validation script estimates quest duration using these heuristics per step type:

| Step Type | Estimated Duration |
|-----------|-------------------|
| `dialogue` | ~30 seconds per 50 words |
| `travel` | ~2 minutes |
| `encounter` | ~5 minutes |
| `fetch` | ~3 minutes |
| `escort` | ~3 minutes |
| `investigate` | ~3 minutes |
| `puzzle` | ~4 minutes |

The total estimated duration is compared against the quest's `estimatedMinutes` field. If the deviation exceeds 50%, the validation script emits a warning. This helps catch quests where the declared duration is wildly different from what the content suggests.

## Balance Constraints

### Difficulty Scaling

Quests near the start of the road (low `distanceFromStart`) should be easier. Guideline:

| Road Segment | Distance Range | Encounter Difficulty |
|-------------|----------------|---------------------|
| Early road | 0 - 8,000 | 1-3 |
| Mid road | 8,000 - 18,000 | 3-6 |
| Late road | 18,000 - 30,000 | 5-10 |

### Reward Progression

- **Micro quest rewards**: Small items, minor currency, maps
- **Meso quest rewards**: Meaningful modifiers, equipment, village reputation
- **Macro quest rewards**: Major unlocks, story progression, powerful equipment

### Quest Duration Guidelines

- Micro quests: 5-15 estimated minutes
- Meso quests: 15-45 estimated minutes
- Macro quests: 30-120 estimated minutes

### Quest Density Target

- 1-2 micro quests per 1,000 world units of road
- 1 meso quest per anchor point
- Macro quests tied to main story chapters
