# Hidden-behavior audit

Log of real bugs I've found by verifying runtime behavior (not just CI green).
Anything below is a confirmed defect with evidence.

## 1. Silent CI skip of browser tests (FIXED — #145)

**Symptom**: Every PR reported green "Test & Lint" but the browser-mode step showed `Tests: no tests` and an unhandled `Executable doesn't exist` error. Had been silently skipping in CI for months.

**Root cause**:
- `.github/workflows/ci.yml`: no `pnpm exec playwright install` step before `pnpm test:browser`
- Same workflow: `continue-on-error: true` on the browser-tests step swallowed the failure

**Evidence**: gh run view of run 24639148571, step "Browser smoke tests" showed `browserType.launch: Executable doesn't exist at /home/runner/.cache/ms-playwright/...`

**Fix**: Install playwright step added; `continue-on-error` removed.

## 2. PBR displacement map loaded then zeroed (FIXED — #146)

**Symptom**: `src/assets/pbr/loader.ts:102` explicitly set `mat.displacementScale = 0.0` immediately after loading the displacement texture. Every PBR asset paid for an unused texture fetch.

**Fix**: Now 0.02 scale with -0.01 bias.

## 3. PBR AO map bound but never sampled (FIXED — #146)

**Symptom**: MeshStandardMaterial.aoMap samples from `uv2`, not `uv`. RoadSurface built geometry with only `uv`, so the AO texture applied to zero pixels.

**Evidence**: `app/scene/RoadSurface.tsx:395` only had `setAttribute('uv')`.

**Fix**: `applyPbrMaterialToMesh()` helper + RoadSurface now sets both uv and uv2.

## 4. Silent catch-and-fallback everywhere (FIXED — #144)

**Symptom**: Environment.tsx had `try { biome = ... } catch { /* use defaults */ }` blocks that swallowed real errors and continued rendering with fallback colors. Any BiomeService bug would be invisible.

**Fix**: Replaced with descriptive throws + `reportRuntimeError()` bus that surfaces via `<ErrorOverlay>`.

## 5. 5 of 6 browser tests had never worked (FIXED — #145)

**Symptom**: LoadingOverlay, DialogueBox, InventoryScreen, MainMenu, QuestLog — all imported Koota `useFlags()` / `useTrait()` without being wrapped in a `<WorldProvider>`. Every attempted run would have thrown `"Koota: useWorld must be used within a WorldProvider"` — but because browser-mode never actually ran on CI (bug #1), nobody noticed. ErrorOverlay's `getByText(/bridge/i)` query also timed out at 60s due to an overflow-auto container confusing the locator.

**Fix**: Added `app/__tests__/test-utils.tsx` with `<KootaProvider>`; switched ErrorOverlay test to `getByTestId('error-title')`.

## 6. Vitest version drift caused 10-minute deadlocks (FIXED — #145)

**Symptom**: `pnpm test:browser` hung in optimizeDeps for 10+ minutes locally. Vitest warning said "Running mixed versions is not supported and may lead into bugs" — vitest@4.0.18 vs @vitest/browser@4.1.4.

**Fix**: Pinned all @vitest/* to ^4.1.4.

## 7. Benchmark spawned in empty tile (FIXED — #149)

**Symptom**: `?benchmark=thornfield` used hardcoded `BENCHMARK_SPAWN_POSITION = (2, 1.6, 24)` which placed the player in an empty world tile. 60-second benchmark captured blank sky, not scene performance.

**Evidence**: Screenshot at cb=148 showed "BENCH: recording" HUD but no scene geometry. Fixed by switching to `resolveSpawnPosition()` (same as `?spawn=`).

## 8. Benchmark sampler returns avgDrawCalls=1, avgTriangles=1 (FIXED — #152)

**Symptom**: After #149 fix, cb=149 benchmark emitted the summary line but counts were wrong:

```text
[BENCHMARK] p50=13.90ms p95=16.20ms p99=17.60ms avgDraws=1 avgTris=1
```

A real Thornfield scene has hundreds of draw calls and 400k+ triangles per frame.

**Root cause**: Frame priority ordering. `BenchmarkFrameSampler` ran its `useFrame` at the default priority 0, which fires BEFORE `BiomePostProcessing`'s priority-1 composer render. `gl.info.render.calls` was therefore sampled before the composer had actually issued any draw calls that frame. See #17 below — same root cause.

**Fix**: Moved the sampler to `useFrame(cb, 2)` so it runs AFTER the composer. See `src/benchmark/runner.tsx`.

## 9. Fixture tests broke happy-dom CI run (FIXED — pushed to #150)

**Symptom**: `*.fixture.test.tsx` files imported `@vitest/browser/context`. Happy-dom runner picked them up alongside regular unit tests. Error: "vitest/browser can be imported only inside the Browser Mode."

**Fix**: Added `'**/*.fixture.test.{ts,tsx}'` to `vitest.config.ts` exclude.

## 11. WeatherSystem compounds fog distances toward zero (FIXED — #152)

**Initial claim** (agent audit 2026-04-20): `WeatherSystem.tsx:298-304` did `scene.fog.near *= densityFactor` every frame, and the entry asserted this would compound near/far toward zero over minutes of play.

**Actual state** (verified while writing #152): `Environment.Fog` (app/systems/Environment.tsx) re-asserts `scene.fog.near/far` from the biome baseline every frame BEFORE WeatherSystem runs its modulation. The `*=` operator therefore did not compound across frames — Environment wiped the prior scaling each tick. The regression described by the audit never actually occurred in practice.

**Fix shipped anyway** (#152): replaced `*=` with a rebase-on-external-write ref pattern so WeatherSystem is safe even if Environment's per-frame re-assertion is ever refactored to biome-change-only.

## 12. Shadow frustum clips mid-distance at dawn/dusk (FIXED — #157)

**Symptom**: `app/systems/Environment.tsx:180-181` has sun height drop to 10m at `sunY ≈ 0`. Far plane is 180m fixed. At low sun angle, objects at 30-50m horizontal distance can fall outside the shadow camera volume → shadows pop out unpredictably.

**Fix**: Compute `shadowFar = max(180, distance(lightPos, camera) + 60)` dynamically.

## 13. Linear THREE.Fog used where FogExp2 is thematically correct (ACTIVE)

**Symptom**: `app/systems/Environment.tsx:358` creates `new THREE.Fog(color, near, far)` — linear falloff. Thornfield is supposed to be "cold dawn fog" — exponential density reads more atmospheric.

**Fix**: For stormy/foggy weather states, prefer `FogExp2(color, density)`.

## 14. AmbientParticles InstancedMesh count init (LOW)

**Symptom**: `app/systems/Environment.tsx:434` — `<instancedMesh args={[undefined, undefined, PARTICLE_COUNT]}>`. Passing undefined geometry/material may not allocate the matrix buffer correctly in R3F.

## 15. Canvas outputColorSpace not explicitly set (LOW)

**Symptom**: `app/scene/GameScene.tsx` Canvas `gl={}` doesn't set `outputColorSpace`. Default is sRGB. With composer's HalfFloat framebuffer and sRGB final write, we may double-encode gamma.

**Fix**: Add `outputColorSpace: THREE.SRGBColorSpace` explicitly, OR use `LinearSRGBColorSpace` if composer handles the conversion in ToneMappingEffect.

## 16. GlbInstancer never disposes GPU resources (LOW)

**Symptom**: `app/scene/GlbInstancer.tsx:58` — geometry/material pulled from `useGLTF` cache. When the chunk unmounts, no `.dispose()` is called. The cache holds the geometry alive globally so unloading far chunks leaks nothing — BUT if a texture is cloned for per-instance tinting, the clone leaks.

**Fix**: Audit the tint-clone path; add cleanup `useEffect` that disposes the clone on unmount.

## 19. Combat damage applied without any spatial check (FIXED — #155)

**Initial claim** (agent audit 2026-04-20): "Swinging the weapon does nothing. No damage is dealt."

**Actual state** (verified while writing #155): damage WAS being dealt — `EncounterSystem.doPlayerAttack` called `playerAttackNearest()` every `PLAYER_ATTACK_INTERVAL` (0.8s) whenever attack was held, applying damage to the first alive monster regardless of aim or distance. The subsystems were wired: `playerAttack`, HP tracking, loot, XP, summary — all present and working. What was missing was any spatial predicate (raycast, cone-cast, distance check) gating the damage on the player actually aiming at the target. The audit entry was wrong about the symptom; the defect itself was real but different.

**Task #17 "completed" status was still wrong.** It shipped the impact VFX without the hit detection that should conditionally trigger them.

**Fix** (#155): `src/combat/hit-test.ts` — pure `pickMeleeTarget(playerPos, cameraForward, monsters, opts)` returning the id of the monster in a forward cone (default 2.5m range, 45° half-angle), or null. `EncounterSystem.doPlayerAttack` now reads the camera via `useThree`, runs the predicate against `activeEncounter.monsters`, and only calls `playerAttack(hitId, ...)` on a successful hit. Misses play the swing animation but deal no damage.

## 20. Quest progress lost on page refresh (FIXED — #154)

**Symptom**: `completeQuest()` / `advanceQuestStep()` / `activateQuest()` / etc. in `src/ecs/actions/quest.ts` patched the in-memory `QuestLog` trait only — a refresh reset state to whatever was in the last manual save slot.

**Fix**: Added `src/db/autosave.ts` with a debounced `scheduleAutoSave()` that writes slot 0. Durable-mutation quest actions call it after patching. The snapshot assembler is registered at `App.tsx` bootstrap so `autosave` doesn't have to import from `@/ecs/actions/*` (avoids a module cycle with quest.ts). Provider returns `null` when `!gameActive` so pending saves can't overwrite slot 0 during reset/load teardown.

## 21. Background tab accumulates delta, player teleports on resume (FIXED — #153)

**Symptom**: `useFrame` keeps calling PlayerController while the tab is backgrounded. Koota state ticks forward. When the user returns, the accumulated velocity snaps the player forward.

**Fix**: Added a `tabHidden` field on `GameFlags` and a `visibilitychange` listener in `Game.tsx` that mirrors `document.hidden` into it. PlayerController + WeatherSystem short-circuit when `tabHidden` is true. Kept distinct from the existing `paused` flag so alt-tabbing doesn't open the pause menu.

## 22. Terrain splat-blend uses roughness-darken AO approximation (KNOWN GAP)

**Status**: audited 2026-04-20, documented but deferred. Fixing requires packing AO+roughness+metalness into a single ORM texture per layer to stay under the 16-unit WebGL2 texture budget.

**Where**: `app/scene/terrain/SplatBlendMaterial.ts:170` — line does `mix(1.0, 0.75, rough)` instead of sampling real AO.

## 23. Relic pickup DOM overlay leaks on chunk unload (FIXED — #198)

**Symptom**: `Relic.tsx` imperatively creates two DOM nodes (radial flash + "+1 Ancient Relic" notification) on collection and fades them via nested setTimeouts. If the chunk unloads during the fade, the nodes and their timers survive.

**Fix**: Track nodes and timers in a ref; add an unmount-only cleanup effect that clears pending timers and removes any still-attached nodes. Same pattern as LoadingOverlay fade-out fix (#193).

## 24. OceanPlane never disposes PlaneGeometry + ShaderMaterial (FIXED — #199)

**Symptom**: `OceanPlane` allocates a 2000×2000 / 64-segment PlaneGeometry (≈16k verts) and a ShaderMaterial via `useMemo` but never releases them. End-session → New-Game leaks them for the tab lifetime.

**Fix**: Unmount-only dispose effect tied to both refs.

## 25. RoadSurface leaks merged geometry + cloned material per chunk (FIXED — #200)

**Symptom**: Biggest R3F GPU leak on a normal session. Chunk unloads (player walks past view-distance) leaked a merged BufferGeometry per chunk, and `roadType` changes swapped in a new material without releasing the previous clone's textures.

**Fix**: Added `disposeRoadMaterial` helper (disposes cloned material + its cloned map/normalMap/roughnessMap — NOT the shared PBR cache's textures) and two dispose effects — one tied to `geometry`, one tied to `material` state.

## 26. Combat VFX pool geometries + SDF materials leaked per encounter (FIXED — #202)

**Symptom**: `ImpactDeformerEffect` (4-pool) and `BloodMetaballsEffect` (3-pool) allocate a hull SphereGeometry + pooled raymarch materials but never release them. EncounterSystem unmounts these at the end of every encounter; GPU resources accumulated.

**Fix**: Unmount-only dispose effects on each component releasing geometry + every pooled material.

## 27. DungeonRenderer leaks per-room wall material clones (FIXED — #203)

**Symptom**: `DungeonRoomMesh` memoizes a tinted clone of the shared `wallMaterial` per room. Every room the player visits leaked its material for the tab's lifetime.

**Fix**: Dispose effect on `[tintedWallMaterial]` that runs on swap or unmount. Shared module-level `wallMaterial` is intentionally NOT disposed.

## 28. Save-db debounce loses writes on tab close (FIXED — #205)

**Symptom**: `save-db.ts` debounces sql.js → IndexedDB writes by 200ms. Closing the tab within that window (quick Cmd-W after a save, or Capacitor backgrounding the WebView) means the debounced setTimeout never fires and the write is lost.

**Fix**: Register a one-time module-load listener for `visibilitychange → hidden` (reliable across desktop + mobile) and `pagehide` (older Safari fallback) that force-flushes any pending writes immediately.

## 29. KeyboardMouseProvider ghost keys on alt-tab (FIXED — #207)

**Symptom**: Player alt-tabs while holding W or left-mouse; the corresponding keyup/mouseup fires on the newly-focused app. `keysDown` keeps the key marked pressed; on return the player resumes walking without touching anything.

**Fix**: Window `blur` listener clears `keysDown` and `attackHeld` on focus loss.

## 30. Audio layers leak Tone nodes on AudioSystem unmount (FIXED — #210)

**Symptom**: AudioSystem called `layer.stop(); layer.gain.dispose()` on teardown — stopping only the source clock and freeing only the gain. Tone.Noise, Tone.Filter, Tone.FMSynth, Tone.Tremolo etc. stayed wired into the AudioContext graph.

**Fix**: Extended `AudioLayer` with a `dispose()` contract that releases every Tone node the factory allocated. AudioSystem calls `layer.dispose()` instead of the partial teardown. Added per-layer dispose-contract tests.

## 31. collectGem missing autosave (FIXED — #212)

**Symptom**: `collectGem` in `src/ecs/actions/game.ts` mutates durable state (chunkDeltas + gemsCollected) but never scheduled an autosave. Picking up a relic → refreshing the tab → relic back, counter reset. Also non-idempotent: useFrame-driven proximity check could re-enter before React flushed `setIsCollected(true)`.

**Fix**: Added `scheduleAutoSave()` call after the mutation + early-return guard when `gemId` is already in the chunk's gems list.

## 32. Dungeon state mutations missing autosave (FIXED — #214)

**Symptom**: Save schema persists active dungeon (id, currentRoomIndex, overworldPosition+Yaw), but `enterDungeon`/`exitDungeon`/`moveToRoom` don't autosave. Entering a dungeon then closing the tab drops the player back in the overworld on reload; room progress within a dungeon is lost.

**Fix**: `scheduleAutoSave()` after each of the three durable mutations.
